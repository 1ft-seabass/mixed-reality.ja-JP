ほとんどのエミュレーターのユーザーがのみの基本的な入力コントロールを使用する必要があります、 [HoloLens のエミュレーター](using-the-hololens-emulator.md#basic-emulator-input)または[Windows Mixed Reality シミュレーター](using-the-windows-mixed-reality-simulator.md#basic-simulator-input)します。 以下の詳細より複雑な種類の入力をシミュレートする必要がある高度なユーザーはいます。

## <a name="concepts"></a>概念

HoloLens のエミュレーターおよび Windows Mixed Reality シミュレーターへの仮想入力を制御する開始するには、まずいくつかの概念を理解する必要があります。

モーションは、制御して、位置と向きのシーンで何らかの変更を表します。 制御可能なオブジェクトを対象となるは、モーションは回転と 3 つの軸に沿った変換 (移動) の両方で制御されます。
* **ヨー**:左または右に有効にします。
* **ピッチ**:アップまたはスケール ダウンを有効にします。
* **ロールバック**:サイド バイ サイドをロールバックします。
* **X**:左または右に移動します。
* **Y**:上下に移動します。
* **Z**:前方または後方に移動します。

[ジェスチャ](gestures.md)とアニメーション コント ローラーの入力は、方法に密接にマップする、物理デバイス。
* **アクション**:これはつまみを人差し指を押すか、コント ローラーのアクション ボタンのプルの操作をシミュレートします。 たとえば、アクションの入力は、コンテンツ、およびプレス アンド ホールドをスクロール、エア タップ ジェスチャをシミュレートするために使用できます。
* **[「Bloom](gestures.md#bloom)またはホーム**:HoloLens bloom ジェスチャまたはコント ローラーの [ホーム] ボタンを使用して、シェルに戻るにし、システムの操作を実行します。

手では、HoloLens V2 に豊富な reprresentation があります。  手には、追跡/未追跡との使用可能な運転のジェスチャだけでなく、思い起こさせますスケルトン モデルそれらに合わせるし、開発者に公開されているようになりましたがあります。  これには、各ハンドの 20 の追跡ポイントが導入されています。  
* **共同**:指定した履歴手の形の 20 の追跡位置の 1 つ。 ポイントが関連付けられている 3d 空間には。
* **発生し得る**:追跡対象の手の関節のすべての完全なコレクション。 現時点では、これは、20 の関節のコレクションです。 

この時点で私たちはエミュレーター UI を個別に各複合的な位置を直接制御を公開しません。 ただしシミュレーション API を使用してそれらを設定する場合があります。 代わりに、一連のエミュレーターを使用するとの間で切り替える便利なの代表的なことがあります。

入力のシミュレートされたセンサーの状態を制御することもできます。
* **リセット**:これは、既定値にすべてのシミュレートされたセンサーを返します。
* **追跡**:位置指定の追跡モードを切り替えます。 たとえば、次のようなアニメーションや効果を作成できます。
  * **既定**:OS は、システムの要求に基づいて最適な追跡モードを選択します。
   * **印刷の向き**:強制的に方向専用の追跡、システムの要求に関係なく。
   * **位置指定**:システムの要求に関係なく、強制的に位置指定の追跡が行われます。
