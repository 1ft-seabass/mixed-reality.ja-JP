# <a name="advanced-hololens-emulator-and-mixed-reality-simulator-input"></a>Advanced HoloLens Emulator と Mixed Reality シミュレーター入力

ほとんどのエミュレーターユーザーは、 [HoloLens エミュレーター](using-the-hololens-emulator.md#basic-emulator-input)または[Windows Mixed Reality シミュレーター](using-the-windows-mixed-reality-simulator.md#basic-simulator-input)の基本入力コントロールのみを使用する必要があります。 より複雑な種類の入力をシミュレートする必要がある上級ユーザー向けの詳細を以下に示します。


## <a name="concepts"></a>概念

HoloLens エミュレーターと Windows Mixed Reality シミュレーターへの仮想入力の制御を開始するには、まずいくつかの概念を理解しておく必要があります。

モーションとは、シーン内の項目の位置と向きを制御および変更することを指します。 対象の制御可能なオブジェクトの場合、動きは3つの軸に沿って回転と平行移動 (移動) の両方で制御されます。
* **ヨー**:左または右に移動します。
* **ピッチ**:オンまたはオフにします。
* **ロール**:左右にロールを展開します。
* **X**:左または右に移動します。
* **Y**:上または下へ移動します。
* **Z**:前方または後方に移動します。

[ジェスチャ](gestures.md)とモーションコントローラーの入力は、物理デバイスの方法に厳密にマップされます。
* **アクション**:これは、人差し指でを thumb に押すか、コントローラーのアクションボタンをプルする操作をシミュレートします。 たとえば、アクションの入力を使用して、エアタップのジェスチャをシミュレートしたり、コンテンツをスクロールしたり、プレスアンドホールドを行うことができます。
* **[ブルーム](gestures.md#bloom)/System のジェスチャまたはホーム**:HoloLens ブルーム/システムジェスチャまたはコントローラーの [ホーム] ボタンは、シェルに戻り、システムアクションを実行するために使用されます。

HoloLens 2 には豊富な reprresentation があります。  追跡と追跡が行われず、ジェスチャにも使用できるようになっただけでなく、手によるスケルトンモデルに適合し、開発者に公開されるようになりました。  これにより、各ハンドで26個の追跡ポイントが導入されます。  
* **ジョイント**:特定の追跡対象に対して、20個の追跡位置のうちの1つ。 これは、ポイントが3d 空間に関連付けられています。
* **危険**:追跡対象の手に含まれるすべての関節の完全なコレクション。 現時点では、これは26個の関節のコレクションです。 

現時点では、シミュレーション API を使用して設定することもできますが、エミュレーターのユーザーインターフェイスを使用して各ジョイントの位置の直接制御を公開しません。 代わりに、エミュレーターでの切り替えを可能にする、便利な代表的な方法が用意されています。

シミュレートされたセンサー入力の状態を制御することもできます。
* **リセット**:これにより、シミュレートされたすべてのセンサーが既定値に戻ります。  HoloLens 2 エミュレーター以降では、適切な修飾子キーまたはボタン (左または右の Alt、またはゲームパッドの左または右のバンパー) を使用して目的のハンドを使用して、1つまたは両方にリセットを適用できます。
* **追跡**:位置追跡モードを順番に切り替えます。 この機能には、次が含まれます。
  * **既定**:OS は、システムに対して行われた要求に基づいて、最適な追跡モードを選択します。
   * **方向**:システムで行われた要求に関係なく、方向のみの追跡を強制的に実行します。
   * **位置**:システムで行われた要求に関係なく、位置追跡を強制的に実行します。

## <a name="targeting"></a>ターゲット設定 

上記の入力概念の一部は、独自に作成されています。  Action、ブルーム/System、Reset、および Tracking は完全な概念であり、必要ではなく、対象となる追加の修飾子もありません。  ただし、残りの概念は、複数のターゲットのいずれかに適用できます。 コマンドを適用する対象となるターゲットを指定する方法が導入されました。  どのような場合でも、UI またはキーボードを使用して、targtet するオブジェクトを指定できます。  場合によっては、xbox コントローラーで直接を指定することもできます。 

次の表では、ターゲット設定のオプションと、各オプションをアクティブ化する方法について説明します。

| オブジェクト | キーボード修飾子 | コントローラー修飾子 | Emulator UI 修飾子 |
|----------|----------|----------|----------|
| 本文 | (既定値)。 | (既定値)。 | (既定値)。 |
| 矢印 | 保持時間 | (利用不可) | (利用不可) |
| 左側/コントローラー | 左に押す Alt ボタン | 左ショルダーボタンを保持する | 左プッシュピン | 
| 右手/コントローラー | 右 Alt ボタンを押す | 右ショルダーボタンを保持する | 右プッシュピン |
| 目 | Y を保持する | (利用不可) | 目の画鋲 |

## <a name="perception-simulation-control-panel-keyboard-shortcuts"></a>認識シミュレーションコントロールパネルのキーボードショートカット

次のキーボードショートカットを使用すると、コントロールパネルの知覚シミュレーションにアクセスし、シミュレーションで使用する PC 入力デバイスを有効または無効にすることができます。

| 操作 | Shortcut | 説明/メモ |
|-----------|----------|-------------|
| [シミュレーションにキーボードを使用] を切り替えます | F4 | オフにすると、キーボード入力が HoloLens または Windows Mixed Reality アプリケーションに送られます。 |
| [シミュレーションにマウスを使用] を切り替えます | F5 | オフにした場合、マウス入力は混合現実環境に移動します (Windows Mixed Reality のみ)。 |
| [シミュレーションにゲームパッドを使用する] を切り替えます | F6 | オフにすると、ゲームパッドの入力はシミュレーションによって無視されます |
| コントロールパネルの表示と非表示を切り替える | F7 | |
| キーボードフォーカスをコントロールパネルに設定する | F8 | パネルが現在表示されていない場合は、最初に表示されます。 |
| エミュレーターまたは Mixed Reality ポータルウィンドウとの間でパネルをドッキングまたはドッキング解除する | F9 | ドッキング解除時にウィンドウを閉じると、ドッキングされて非表示になります。 |
