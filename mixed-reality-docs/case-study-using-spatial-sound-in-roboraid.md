## <a name="do-it-yourself"></a>自分で行う

(爆発) などのイベントをより大きな音をことを確認する検出は 1 つのトリック: ルームを入力できるように、空間のサウンドの mono の資産を作成し、3 D で再生するための 2D ステレオ資産と併用することでした。 かかりますチューニングによって、ステレオのコンテンツで大量の情報を持つ mono の資産の方向性を抑えられるためです。 ただし、適切なバランスを取得する、正しい方向に頭を有効にするプレーヤーを取得する膨大なサウンドされます。

次のオーディオ アセットを使用して自分で試すことできます。

**シナリオ 1**
1. ダウンロード[roboraid_enemy_explo_mono.wav](images/roboraid-enemy-explo-mono.wav)で空間のサウンドの再生に設定され、イベントに割り当てるとします。
2. ダウンロード[roboraid_enemy_explo_stereo.wav](images/roboraid-enemy-explo-stereo.wav) 2D ステレオで再生に設定して、上記と同じイベントに割り当てます。 Unity には、これらの資産が正規化され、ためには、クリップにしないように両方のアセットのボリュームを減衰させます。
3. 両方のサウンドを一緒に再生します。 空間がどのように聞こえると思われるために、頭に移動します。

**シナリオ 2**
1. ダウンロード[roboraid_enemy_explo_summed.wav](images/roboraid-enemy-explo-summed.wav)で空間のサウンドの再生に設定され、イベントを割り当てるとします。
2. この資産を単独で再生し、シナリオ 1 からのイベントを比較します。
3. Mono とステレオのファイルの別の分散を実行してください。
