* <span data-ttu-id="a6832-172">カーソルを使用して、ユーザーの入力状態を表示できます。</span><span class="sxs-lookup"><span data-stu-id="a6832-172">We could use the cursor to display the user's input state.</span></span> <span data-ttu-id="a6832-173">たとえば、システムに手の状態が表示された場合にユーザーに通知するアイコンを表示することができます。</span><span class="sxs-lookup"><span data-stu-id="a6832-173">For example, we could display an icon telling the user if the system sees their hand state.</span></span> <span data-ttu-id="a6832-174">これにより、ユーザーにアクションを実行する準備ができたことがユーザーに通知されます。</span><span class="sxs-lookup"><span data-stu-id="a6832-174">This will tell the user that the application knows the user is ready to take action.</span></span>
* <span data-ttu-id="a6832-175">また、カーソルを使用して、音声コマンドが使用可能であることをユーザーに認識させることもできます。</span><span class="sxs-lookup"><span data-stu-id="a6832-175">We could also use the cursor to make the user aware that there is a voice command available.</span></span> <span data-ttu-id="a6832-176">または、カーソルの色を一時的に変更して、システムによって音声コマンドが通知されたことをユーザーに知らせます。</span><span class="sxs-lookup"><span data-stu-id="a6832-176">Or perhaps change the color of the cursor momentarily to tell the user that the voice command was heard by the system.</span></span>

<span data-ttu-id="a6832-177">これらのさまざまなカーソルの状態は、さまざまな方法で実装できます。</span><span class="sxs-lookup"><span data-stu-id="a6832-177">These different cursor states can be implemented in different ways.</span></span> <span data-ttu-id="a6832-178">ステートマシンのようにカーソルをモデル化することによって、これらの異なる状態を実装する場合があります。</span><span class="sxs-lookup"><span data-stu-id="a6832-178">You might implement these different states by modeling the cursor like a state machine.</span></span> <span data-ttu-id="a6832-179">以下に例を示します。</span><span class="sxs-lookup"><span data-stu-id="a6832-179">For example:</span></span>
* <span data-ttu-id="a6832-180">アイドル状態は、既定のカーソルを表示する場所です。</span><span class="sxs-lookup"><span data-stu-id="a6832-180">Idle state is where you show the default cursor.</span></span>
* <span data-ttu-id="a6832-181">準備完了の状態は、ユーザーの手が準備完了の位置にあることを検出したときに表示されます。</span><span class="sxs-lookup"><span data-stu-id="a6832-181">Ready state is when you have detected the user's hand in the ready position.</span></span>
* <span data-ttu-id="a6832-182">対話状態は、ユーザーが特定の操作を実行しているときに行われます。</span><span class="sxs-lookup"><span data-stu-id="a6832-182">Interaction state is when the user is performing a particular interaction.</span></span>

<span data-ttu-id="a6832-184">また、これらの状態は、さまざまな状態が検出されたときに異なるアートアセットを表示できるように、スキン対応の方法で実装することもできます。</span><span class="sxs-lookup"><span data-stu-id="a6832-184">You could also implement these states in a skin-able manner such that you can display different art assets when different states are detected.</span></span>

## <a name="going-cursor-free"></a><span data-ttu-id="a6832-185">"カーソルを利用しない"</span><span class="sxs-lookup"><span data-stu-id="a6832-185">Going "cursor-free"</span></span>

<span data-ttu-id="a6832-186">カーソルを使用せずにデザインすることをお勧めするのは、immersion の意味がエクスペリエンスの重要なコンポーネントであり、(宝石とジェスチャを通じて) ターゲット設定を伴う相互作用には、優れた精度を必要としない場合のみです。</span><span class="sxs-lookup"><span data-stu-id="a6832-186">Designing without a cursor is recommended only when the sense of immersion is a key component of an experience, and interactions that involve targeting (through gaze and gesture) do not require great precision.</span></span> <span data-ttu-id="a6832-187">システムは、カーソルによって通常満たされているニーズを満たす必要があります。ただし、ユーザーは、システムの対象についての理解に関する継続的なフィードバックを提供し、システムに対する意思を確実に伝えるために役立ちます。</span><span class="sxs-lookup"><span data-stu-id="a6832-187">The system must still meet the needs that are normally met by a cursor though - providing users with continuous feedback on the system's understanding of their targeting and helping them to confidently communicate their intentions to the system.</span></span> <span data-ttu-id="a6832-188">これは、他の顕著な状態の変更によって実現可能です。</span><span class="sxs-lookup"><span data-stu-id="a6832-188">This may be achievable through other noticeable state changes.</span></span>
